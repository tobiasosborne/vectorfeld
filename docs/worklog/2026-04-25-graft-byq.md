# 2026-04-25 — Graft Phase 2 starts: vectorfeld-byq

First of the six implementation beads under epic `vectorfeld-ccl` (graft-based PDF round-trip). Adds the source-PDF byte store to `DocumentState` so the graft engine (`vectorfeld-wjj`, next bead) has bytes to work with.

## What shipped

- **`src/model/sourcePdf.ts`** (new, ~90 LOC) — `SourcePdfStore` class with a `primary` slot + a `backgrounds: Map<layerName, …>` for `Open as Background Layer…` imports. Each `SourcePdfEntry` is `{ bytes: Uint8Array; filename: string; pageCount: number; mupdfHandle?: unknown }`. The `mupdfHandle` field is reserved for the worker keep-alive protocol (later bead). Includes the same active-pointer pattern (`setActiveSourcePdfStore`, `getActiveSourcePdfStore`) as the other state modules + a pure `recordImportedSource(store, kind, layerName, entry)` helper.

- **`src/model/sourcePdf.test.ts`** (new) — 9 unit tests covering primary set/clear, background add/remove/lookup, store isolation, the active-pointer swap, and the kind dispatch (primary clears all; background needs layerName).

- **`src/model/documentState.ts`** — added `sourcePdf: SourcePdfStore` field. Wired into `setActiveDocument` and `captureActiveDocumentState` so multi-doc work later swaps it atomically alongside selection/grid/guides/etc.

- **`src/model/documentState.test.ts`** — added isolation test (`sourcePdf` independent across two `DocumentState` instances) and active-swap test (a→b→a round-trips primary entries with byte-array identity).

- **`src/model/pdfRender.ts`** — added `renderPdfPageToSvgWithMeta(pdf, pageIndex)` returning `{ svg, pageCount }`. Existing `renderPdfPageToSvg` kept as a thin wrapper for back-compat (tests + standalone callers).

- **`src/model/pdfRender.worker.ts`** — protocol now forwards `pageCount` in the rendered response.

- **`src/model/pdfImport.ts`**:
  - Worker protocol: `RenderResult = { svg, pageCount }`. Pending callbacks updated.
  - Both import paths (`importPdf`, `importPdfAsBackgroundLayer`) now read the file once, fork the bytes via `slice(0)` so a transferable can still go to the worker without detaching the retained copy.
  - After successful render: `recordImportedSource(getActiveSourcePdfStore(), kind, layerName, entry)` lands the bytes + filename + pageCount in the active doc's store.

- **`test/golden/stories/05-pdf-roundtrip.mjs`** — fixture path corrected from `temp/` to `test/dogfood/fixtures/` (consequence of the prior tidy).

## What this unblocks

`vectorfeld-wjj` (new `src/model/graftExport.ts` engine) — can now read the source bytes off `getActiveSourcePdfStore().getPrimary()` and call `mupdf.PDFDocument.graftPage()` per spike-01 (proven byte-for-byte fidelity, 0.0000% pixel diff). When the engine wires up the worker keep-alive protocol it can populate the `mupdfHandle` field; for now bytes alone are enough.
