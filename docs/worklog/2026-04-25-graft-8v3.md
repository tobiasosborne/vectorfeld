# 2026-04-25 — Graft Phase 2: source-PDF back-references on SVG elements (vectorfeld-8v3)

Second of the six implementation beads under `vectorfeld-ccl`. Tags every imported leaf graphical element with `data-src-page` + `data-src-layer-id` so the graft engine (next bead, `wjj`) can tell at flush time which subtrees are unmodified source content (graft byte-for-byte) versus user-authored or mutated content (overlay-render).

## What shipped

- **`src/model/sourceTagging.ts`** (new, ~95 LOC) — `tagImportedLayer(layer, { page, layerId })` recursively tags `<text>`, `<tspan>`, `<path>`, `<image>`, `<rect>`, `<circle>`, `<ellipse>`, `<line>`, `<polygon>`, `<polyline>` with `data-src-page` and `data-src-layer-id`. Containers (`<g>`, `<defs>`, `<title>`, `<desc>`, `<metadata>`) are recursed through, never tagged. The IMPORTED LAYER itself gets `data-source-pdf-id` — a stable key that survives the user renaming the layer in the Layers panel. Also exports `getSourceMeta`, `isFromSource`, `getLayerSourceId`, and `lookupSourceEntry(store, layerId)` which resolves a layer-id back to its `SourcePdfStore` entry (`PRIMARY_LAYER_ID` → `getPrimary()`, anything else → `getBackground(layerId)`).

- **`src/model/sourceTagging.test.ts`** (new) — 13 unit tests: stable layer-id, every taggable tag gets stamped, containers don't, recursion through nested groups, existing attributes preserved, `getSourceMeta` round-trips, `isFromSource` predicate, `lookupSourceEntry` for primary + backgrounds + missing.

- **`src/model/pdfImport.ts`** — both import paths (`importPdf`, `importPdfAsBackgroundLayer`) now call `tagImportedLayer` after `flattenAndScalePdfLayer` + `tagLayerWithImportAnalysis`. Primary-import children get `layerId: PRIMARY_LAYER_ID` (sentinel `__primary__`). Background-import children get `layerId: <sanitized-filename>` matching the `SourcePdfStore.backgrounds` Map key.

- **`test/golden/canonicalize.mjs`** — strip list extended with `data-src-page`, `data-src-layer-id`, `data-source-pdf-id`. These are editor-internal back-references, not part of the semantic content the user sees in exported files. Golden masters didn't need re-recording.

## Deferred to Phase 3

The bead description called for `data-src-op-offset` (byte offset into the source page's content stream where the originating Tj/TJ op starts). This requires:
- A custom content-stream tokenizer (~120 LOC, see `docs/spikes/spike-03-findings.md`)
- Per-font ToUnicode CMap decoding (~100 LOC)
- A correlation strategy between SVG `<text>` elements and source ops (positional or content-based)

Total: ~250 LOC, well past `8v3`'s budget. The graft engine (`wjj`, next) doesn't need op-offsets — it only needs to know "is this from source?" (page + layer-id is enough). Op-offsets are the Phase 3 territory: in-place source-text edits via `vectorfeld-eb0`.
