# 2026-04-25 — Graft Phase 2: command-level source-touch classification (vectorfeld-5gk)

Third of the six implementation beads under `vectorfeld-ccl`. Adds `touchesSource()` to every command in `commands.ts` so the graft engine (next bead, `wjj`) can walk the command history and decide per-element: graft byte-for-byte vs. re-render via overlay content stream.

## What shipped

- **`src/model/commands.ts`**:
  - `Command` interface gains an optional `touchesSource?(): boolean`. Defaults to false (the safe choice — overlay-render is always correct, just slower than graft).
  - Helper `commandTouchesSource(cmd: Command): boolean` reads the method safely with a false fallback for legacy command-shaped objects in tests.
  - Implementations:
    - `AddElementCommand` — always `false` (introduces new untagged elements).
    - `RemoveElementCommand` — `isFromSource(this.element)`.
    - `ModifyAttributeCommand` — `isFromSource(this.element)`.
    - `ReorderElementCommand` — `isFromSource(this.element)`.
    - `GroupCommand` — any of `this.children` `isFromSource()`.
    - `UngroupCommand` — any of `this.children` `isFromSource()`.
    - `CompoundCommand` — `this.commands.some(commandTouchesSource)`.

- **`src/model/commands.test.ts`** — 7 new tests covering each command's classification, plus `commandTouchesSource`'s fallback for legacy mocks.

## Why optional

Many existing test files declare inline command-shaped objects (`{ description, execute, undo }`) that flow through `CompoundCommand` and `CommandHistory`. Making `touchesSource` required would force every one of those mocks to add `touchesSource: () => false`. Optional + helper is far less churn. The public `commandTouchesSource(cmd)` accessor is the canonical read site so the default is enforced exactly once.

## No behavior change

This bead only adds the metadata. Nothing reads `touchesSource()` yet — that wires up in `wjj` (the graft engine) and `u7r` (the export wire-in).

## Next work

`vectorfeld-wjj` — the engine itself. ~250 LOC:
- Take a `DocumentModel`, walk every layer.
- For each layer with `data-source-pdf-id`: look up the `SourcePdfStore` entry, open its bytes via mupdf, `graftPage()` into a fresh output document.
- Walk the layer's tagged children. Cross-reference with the command history (filtered by `commandTouchesSource(cmd)`) to decide which subtrees to graft vs. re-render.
- For touched elements: append an overlay content stream that re-renders just those elements over the grafted source.
- Save bytes via `mupdf.PDFDocument.saveToBuffer('compress=yes')` and return.

Spikes have already proven the primitives (`docs/spikes/spike-01-verdict.md` for graft-page byte fidelity, `spike-02-verdict.md` for content-stream overlay).
